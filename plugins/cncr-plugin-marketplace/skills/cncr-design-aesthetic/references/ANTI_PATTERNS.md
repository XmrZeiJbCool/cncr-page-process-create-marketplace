# ANTI_PATTERNS

## 作用
- 本文档定义 `pencli MCP` 设计预览阶段必须规避的常见 AI 设计坏味道。
- 进入设计阶段时，应先阅读本文件，再生成页面预览。

## 必避反模式
- 禁止未确认画布尺寸就直接开始摆放组件。
- 禁止把查询页、仪表盘页画成居中窄栏。
- 禁止主内容区四周留白过大，导致页面像未铺开。
- 禁止卡片区、表格区、操作栏没有容器边界感。
- 禁止所有区块无阴影、无分隔、无层级，像平铺白板。
- 禁止把表格区画成无表头的空矩形。
- 禁止默认塞满假图表数据或假表格数据来伪装完成度。
- 禁止主按钮做成普通灰按钮或无层级按钮。
- 禁止所有标题、正文、说明文字使用同一字号和粗细。
- 禁止三列布局画成宽度不稳定、左右不齐或高低错落明显的结构。
- 禁止只有图表框，没有标题区、状态区或容器边界。
- 禁止工具栏与内容区连成一片，没有独立层级。
- 禁止因为没有 `.pen` 组件就用普通方块敷衍真实组件占位。
- 禁止在用户已确认设计预览后，实现阶段随意改布局结构或主要视觉方向。

## 输出要求
- 若设计预览阶段命中以上任一反模式，应主动修正后再输出预览。
- 若最终仍存在无法规避的反模式风险，必须在设计预览说明中显式指出。
