# 阶段7：前台生存测试

## 目标
- 在页面可访问的前提下，对真实前端进行最小生存测试，避免只能靠人工随意点击验证。

## 必须输出
- 测试入口地址
- 测试方式（Claude 自动执行 / 用户手动执行）
- 生存测试结果
- 关键验收结论
- 页面截图或等价证据说明

## 推荐测试路径
1. 打开开发服务器页面。
2. 触发最小查询动作（如“开始查询”按钮）。
3. 观察浏览器控制台与页面状态。
4. 检查样式是否明显偏离业务 MD / 参考图。
5. 输出以下验收项：
   - 页面可访问性：成功OK / 失败FAIL
   - API参数完整性：成功OK / 失败FAIL
   - 权限链路验证：成功OK / 失败FAIL
   - `headers + data` 结构验证：成功OK / 失败FAIL
   - `resultData` 归一化验证：成功OK / 失败FAIL
   - 区域级回填日志：成功OK / 失败FAIL
   - 样式还原检查：成功OK / 失败FAIL

## 执行策略
- 若存在可用浏览器自动化能力（如 Claude 当前环境支持前台测试/浏览器测试），优先由 Claude 执行前台生存测试。
- 若当前环境无法自动操作浏览器，则必须输出一份最小人工测试清单，供用户逐项验证，并说明需要回传的关键证据（控制台日志 / 页面截图 / 错误信息）。

## 禁止事项
- 禁止跳过 API 参数完整性与权限链路验证。
- 禁止跳过样式还原检查。
- 禁止只以“页面打开了”作为生存测试通过依据。
