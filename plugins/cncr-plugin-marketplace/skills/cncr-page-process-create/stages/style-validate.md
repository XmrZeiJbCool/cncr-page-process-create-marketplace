# 阶段5：样式还原验收

## 目标
- 在功能可运行基础上，单独验证样式、结构与业务 MD / 参考图 / 预开发设计预览的一致性，避免只完成数据回填却忽略视觉还原。

## 必须输出
- 样式还原结论
- 结构一致性结论
- 主要样式偏差点
- 验证证据来源（页面截图 / DOM结构 / 代码位置）

## 必查项
- 布局结构是否遵循业务 MD 第3章，而不是退化成通用模板
- 若前置阶段已生成 pencli 设计预览图，必须检查实现结果与设计预览是否一致
- 若实现阶段偏离已确认的 pencli 设计预览，必须直接判定为 `样式还原: 失败FAIL`，除非交付中已明确说明偏离原因并获得用户接受
- 关键视觉样式是否对齐：间距、字号层级、颜色、边框、圆角、对齐方式
- 是否符合 `AI_DESIGN_GUIDE.md` 里的默认设计范式：画布铺满、卡片阴影、默认空态、表头可见、主按钮蓝色圆角
- 是否规避了 `ANTI_PATTERNS.md` 中定义的反模式
- 组件层级是否一致：头部 / 图表区 / 表格区 / 操作区 不得随意改层级
- 若存在参考图或既有页面，必须明确说明“已对齐项 / 未对齐项”

## 结论要求
- 不得只输出“样式基本完成”或“样式已处理”这类模糊描述。
- 必须输出：`样式还原: 成功OK` 或 `样式还原: 失败FAIL`。
- 若为 `失败FAIL`，必须列出至少一个具体偏差点，不得在最终交付中写成“实现无误”。
