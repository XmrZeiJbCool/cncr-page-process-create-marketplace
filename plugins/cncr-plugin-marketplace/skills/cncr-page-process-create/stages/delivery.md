# 阶段8：最终交付

## 目标
- 汇总所有阶段结论，形成稳定、可核查的交付报告。

## 必须输出
- 文件落盘清单
- 依赖安装清单
- 阶段验收清单
- 构建与运行结果
- 设计预览确认结果
- Python 静态校验结果
- Middlequery 查询编排结果
- 样式还原验收结果
- 遗留问题 / 已知风险

## 交付硬规则
- 最终交付报告中，必须单独包含 `设计预览确认结果` 分区，不得省略。
- `设计预览确认结果` 至少包含以下两种形态之一：
  - 使用设计预览时：预览方式（`pencli MCP` / `Mermaid fallback`）、预览证据（截图/图示说明）、用户是否已用 `继续开发` 完成确认
  - 跳过设计预览时：`设计预览已跳过（用户选择模式2）`
- 最终交付报告中，必须单独包含 `Python静态校验结果` 分区，不得省略。
- `Python静态校验结果` 至少包含：
  - 执行命令
  - `valid`
  - `issues`
  - `area_fill_summary`
  - `query_middlelayer_detected`
  - `missing_query_middlelayer_fields`
  - `query_middlelayer_error_tokens`
- 若 Python 校验失败，不得在验收清单中写“实现无误”或“回填无误”。
- 最终交付报告中，必须单独包含 `样式还原验收` 分区。
- `样式还原验收` 必须明确输出：`成功OK` 或 `失败FAIL`，并附具体偏差点或对齐说明。
